General User / Driver Instructions

POD Event Stages & Requirements

The system dynamically changes what is required based on the Action Type you select. Always start by scanning or typing the HWB.

  • 1. Shipper Pickup (Leg 1 Start)

    Required: HWB Number, Action Type.

    Optional: Recipient Name, Photo, Signature.

    Action: Scan the freight at the pickup location and submit to mark it "In Progress".

  • 2. Origin Airport Drop (Leg 1 Complete)

    Required: HWB Number, Action Type, Recipient Name.

    Instructions: You must enter the Airline Name or the Receiving Agent into the "Recipient Name" field. Photos and Signatures are optional.

  • 3. Destination Airport Pickup (Leg 3 Start)

    Required: HWB Number, Action Type.

    Optional: Recipient Name, Photo, Signature.

    Action: Scan the freight when recovering it from the destination airport to begin the final mile.

  • 4. Consignee Drop (Final Delivery)

    Required: HWB Number, Action Type, Recipient Name, POD Photo, AND Consignee Signature.

    Instructions: This is the final milestone. The system will reject the submission if the name, photo, or signature are missing.

Off-Sheet Freight Reassignment

If a scanned HWB is currently assigned to another driver/load, the system will show an off-sheet freight warning before allowing any status event to be submitted. Checking the confirmation box confirms you are reassigning that freight to your active workflow. This warning behavior is designed to prevent accidental load theft, duplicate handling, or incorrect custody updates.

Batch Uploads

Use the Upload tab to submit up to 100 scanned documents in one batch. Files are sent to Couchdrop for processing.
