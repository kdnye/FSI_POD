General User / Driver Instructions

POD Event Stages & Requirements

The system dynamically changes what is required based on the Action Type you select. Always start by scanning or typing the HWB.

  • 1. Shipper Pickup (Leg 1 Start)

    Required: HWB Number, Action Type.

    Optional: Recipient Name, Photo, Signature.

    Action: Scan the freight at the pickup location and submit to mark it "In Progress".

  • 2. Origin Airport Drop (Leg 1 Complete)

    Required: HWB Number, Action Type, Recipient Name.

    Instructions: You must enter the Airline Name or the Receiving Agent into the "Recipient Name" field. Photos and Signatures are optional.

  • 3. Destination Airport Pickup (Leg 3 Start)

    Required: HWB Number, Action Type.

    Optional: Recipient Name, Photo, Signature.

    Action: Scan the freight when recovering it from the destination airport to begin the final mile.

  • 4. Consignee Drop (Final Delivery)

    Required: HWB Number, Action Type, Recipient Name, POD Photo, AND Consignee Signature.

    Instructions: This is the final milestone. The system will reject the submission if the name, photo, or signature are missing.

Off-Sheet Freight Reassignment

If a scanned HWB is currently assigned to another driver/load, the system will show an off-sheet freight warning before allowing any status event to be submitted. Checking the confirmation box confirms you are reassigning that freight to your active workflow. This warning behavior is designed to prevent accidental load theft, duplicate handling, or incorrect custody updates.

Batch Uploads

Use the Upload tab to submit up to 100 scanned documents in one batch. Files are sent to Couchdrop for processing.

Operations Instructions

Load Board Filters & Visibility

Use the toggles at the top of the Active Load Board to adjust visibility:

  • Show Delivered: Displays loads that have reached "Consignee Drop" (final delivery). By default, these hide after 4 hours to keep the board clean.
  • Show Cancelled: Reveals loads that were manually cancelled or soft-deleted.

Understanding Workflow Stages

The Load Board tracks physical progression across three legs. The UI automatically advances as drivers log POD events:

  • Awaiting Pickup / En Route: The Pickup Driver is actively assigned to Leg 1.
  • At Origin Airport (In Transit): Leg 1 is complete. The freight is "in the air" (Leg 2). The board pivots to display the Delivery Driver assigned to Leg 3.
  • Out for Delivery: The freight has been recovered at the destination airport, and the Delivery Driver is heading to the Consignee.
  • Delivered: Final delivery is complete.

CSV Uploads (Source of Truth)

Ops/Admin CSV uploads require the following headers: Mawb#, HWB, Org, Dest, PU Driver, DEL Driver.

Warning: Uploading a CSV forcefully overwrites existing driver assignments and leg statuses. To prevent data drift, do not upload a stale CSV if drivers have already made physical progress on those loads.

Live Dashboard & POD Exports

Use the Live Ops Dashboard to monitor real-time sync, or download the global POD history CSV from the POD History export section.

Administrator Instructions

System Overrides & Load Clearance

Admins can manually override shipment statuses using the Resolve buttons on the Load Board.

  • CANCEL: Soft-deletes the load (moves it to a Cancelled state).
  • 3RD PARTY: Marks the load as Delivered when completed by an external carrier.

Bulk Clear / Hard Delete

Use the "Clear All Active Loads" button to wipe the board. You will be prompted for a confirmation phrase:

  • Type CLEAR ALL to soft-delete (Cancel) all active shipments.
  • Type HARD DELETE to permanently erase all active shipments from the database. Use this to cleanly purge bad test data before a fresh CSV upload.

User Management

In the Admin Dashboard (/account/admin/users), approve pending employees, assign roles (SUPERVISOR, FINANCE, ADMIN), and toggle Ops membership for qualified users.
